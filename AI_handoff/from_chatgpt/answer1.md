🔥 2. Главная ошибка сейчас (и Deepseek, и многие делают её)

Ты стоишь перед выбором:

circuit breaker
ablation
simpler model
walk-forward

❗ Их нельзя делать в произвольном порядке.

Если сделать не в том порядке → ты закрепишь ложные выводы.

✅ Правильный порядок (жёстко)
🟥 P0 — Circuit breaker (СДЕЛАТЬ СРАЗУ)

Не для качества. Для выживания системы.

У тебя уже есть почти всё в конфиге:

"drift": {
    "min_win_rate": 0.45,
    "min_profit_factor": 1.10,
    ...
}

👉 Но этого мало. Нужно поведение:

Добавить:
fallback → sent_off + minimal model
снижение позиции (например x0.25)
hard disable торговли при:
PF < 0.8
Sharpe < 0
N подряд лоссов

👉 Это не улучшает модель, но спасает депозит.

🟧 P1 — Walk-forward (КРИТИЧНО, ДЕЛАТЬ ПЕРВЫМ ДЛЯ ML)

Copilot здесь абсолютно прав:

это важнее, чем любые гиперпараметры

Почему:

Сейчас ты оптимизируешь:

accuracy ❌
на одном периоде ❌

→ это гарантирует переобучение

Сделай минимально честную схему:
Train:   Apr–May
Valid:   Jun
Test:    Jul–Sep (holdout)

И дальше:

никакой информации из future
выбор модели = по VALID
финальная проверка = TEST

👉 Без этого любые следующие шаги бессмысленны.

🟨 P2 — Feature ablation (САМОЕ ВАЖНОЕ ДЛЯ СИГНАЛА)

Вот где ты реально “лечишь” модель.

Подозрительные группы (по приоритету):
1️⃣ rolling — 🔥 главный кандидат

Почему:

куча окон [5,10,15,30,60,120]
сильно коррелированы
ловят шум
идеально “запоминают” локальный режим

👉 Типичный источник overfit на 15m

2️⃣ sentiment — 🟠 почти наверняка шум

У тебя:

"SENTIMENT_WEIGHT": 0.30

Это огромный вес для сигнала, который:

не синхронизирован по latency
может быть leakage
нестабилен

👉 На holdout он тебя убивает (видно по результатам)

3️⃣ regime — 🟡 зависит от реализации

Если:

он смотрит в будущее (косвенно) → опасно
или слишком сложный → переобучение
✔️ План ablation (очень конкретно)

Запусти 4 модели:

baseline
только price / returns / simple indicators
-rolling
-sentiment
-rolling -sentiment (самый важный тест)

Сравни:

Sharpe (VALID и TEST)
PF
стабильность

👉 Ты почти гарантированно увидишь:

модель без rolling + sentiment будет хуже на June, но лучше на holdout

И это именно то, что нужно.

🟩 P3 — Simplify LightGBM (ПОСЛЕ ablation)

Deepseek прав, но порядок важен.

Почему НЕ раньше:

Если сначала упростить модель:

она всё равно будет учить плохие фичи
просто хуже
Минимальный сет:
num_leaves = 15
max_depth = 4
min_child_samples = 100
можно добавить:
feature_fraction = 0.6
bagging_fraction = 0.6

👉 цель: снизить capacity, не убив сигнал

❓ 3. Sentiment — отключать ли?

👉 Да. На следующий цикл — OFF по умолчанию.

Причины:

holdout не подтверждает value
высокий риск leakage
большой вес (30%)

👉 Вернуть можно только если:

он проходит ablation
даёт стабильность на walk-forward
❗ 4. Самое опасное место в твоей системе

Не модель.

А вот это:

"windows": [5, 10, 15, 30, 60, 120]

Это:

мультиколлинеарность
огромное пространство поиска
идеальный overfit генератор

👉 Сократи до:

[10, 30, 60]

или даже:

[15, 60]
🎯 5. Критерий допуска к paper trading

Сейчас у тебя его нет (это проблема).

Минимальный acceptable target:
На holdout (НЕ на June):
Sharpe > 1.0
Profit Factor > 1.3
Return > 0
Max DD < 5–8%
стабильность по окнам (не один lucky run)
И самое важное:

👉 разница между VALID и TEST должна быть маленькой

Если:

VALID Sharpe = 3
TEST Sharpe = -1

→ модель мусор, даже если “иногда стреляет”
